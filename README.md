# ИИ-анализ продажных звонков (MVP)

Веб-приложение на **Next.js 14 (App Router)** для загрузки аудиозаписи звонка и скрипта продаж, автоматической **транскрибации (Groq Speech-to-Text, `whisper-large-v3`)**, **анализа качества продаж** через LLM (**Groq** или **OpenRouter**) и выгрузки **PDF-отчёта** на русском языке.

Весь пользовательский интерфейй и отчёты — **на русском языке**.

## Возможности

- Загрузка аудио **MP3 / WAV / M4A** (перетаскивание и выбор файла).
- Поле **«Скрипт продаж»** с подсчётом символов и очисткой.
- Кнопка **«Анализировать звонок»** — пошаговый экран ожидания без «сырого» JSON.
- API: `POST /api/transcribe`, `POST /api/analyze`.
- Дашборд: оценка, соблюдение скрипта, ошибки, рекомендации, сильные стороны, возражения, закрытие, транскрипция.
- **Скачать PDF отчет** — многостраничный отчёт с кириллицей (шрифт Noto Sans по CDN).
- Светлая / тёмная тема, адаптивная вёрстка.

## Требования

- **Node.js** ≥ 18.17 (рекомендуется LTS 20+).
- Ключ **GROQ_API_KEY** — для **транскрибации** (Speech-to-Text) и, при желании, для **анализа** через Groq.
- Если анализ без Groq: **OPENROUTER_API_KEY** (транскрибация всё равно требует **GROQ_API_KEY**).

## Настройка API

### Транскрибация (`/api/transcribe`)

- Используется **Groq Speech-to-Text** (`groq-sdk`), модель по умолчанию **`whisper-large-v3`** (переопределение: `GROQ_TRANSCRIPTION_MODEL`).
- Язык: **`ru`**, форматы: **mp3, wav, m4a**; ответ нормализуется до чистого текста (`src/lib/groq-speech-transcribe.ts`).

### Анализ (`/api/analyze`)

- Если задан `GROQ_API_KEY`, запросы идут в **Groq** с `response_format: json_object`.
- Иначе используется **OpenRouter** (`OPENROUTER_API_KEY`).
- Ответ нормализуется и проверяется схемой **Zod** (`src/lib/analysis-schema.ts`).

### Лимиты и продакшен

- На **Vercel** действуют лимиты времени выполнения функции и размера тела запроса; для длинных звонков может понадобиться **Pro** и увеличенный `maxDuration` в `route.ts`.
- Для MVP установлен ориентир **до 25 МБ** на файл (см. `src/app/api/transcribe/route.ts`).
- Аудио **не сохраняется на диск** постоянно: буфер обрабатывается в памяти процесса.

## Деплой на Vercel

1. Репозиторий подключите к Vercel.
2. В **Settings → Environment Variables** добавьте те же переменные, что в `.env.example`.
3. Укажите `OPENROUTER_SITE_URL` как URL продакшен-сайта (для OpenRouter).
4. Задеплойте; при ошибках по таймауту увеличьте лимит плана или разбейте пайплайн на очередь (вне рамок MVP).
